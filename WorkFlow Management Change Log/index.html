Work Flow Management – Release Notes

Version: 1.4.0  •  Release Date: December 2025

I. Fixes

  • Resolved an issue that caused data loss when updating the Request Source within the Service menu.

II. Enhancements

  • Improved the QR code scanning capability to support scanning when the system is accessed in mobile view via a web browser.
  • Enhanced the presentation and consistency of data displayed in the Data Table within popup dialogs triggered after request submission, including the following refinements:
    • Separation of labels and values into clearly defined row-based formats to align with standard text display conventions.
    • Introduction of collapsible and expandable data sections to improve readability and user navigation within complex datasets.

III. New Features

  • Implemented a confirmation prompt when a user attempts to exit the interface during the creation of a new request.
  • Added a new Address control type, enabling users to configure custom address structures for system usage.
  • Introduced the Enter Multiple Request tab within the Service menu, allowing users to create multiple requests for any supported service using a variety of range-based entry types.
  • Added a Batch Request configuration option under the External System section in the Service menu, enabling the setup of multiple requests originating from the WorkFlow SDK or other integrated systems.
  • Implemented the Inbound Create Request feature in the External Integrate menu, providing the capability to configure API endpoints for receiving and processing request-creation operations from external systems.
